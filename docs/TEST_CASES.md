# V1 Manual Test Cases

本文記錄 HSR Damage Lab V1 單次傷害計算的手動測試案例，用來確保未來修改公式、光錐、隊伍 Buff、敵人乘區或 UI 時，可以檢查輸出是否仍合理。

## 1. 文件目的

這份文件用來保存 V1 的人工回歸測試清單。每次修改傷害公式、資料來源、光錐效果、隊伍 Buff、敵人乘區、Formula Debug 或 UI 顯示時，應依照本文件檢查主要輸出是否仍符合目前設計。

目前尚未接入自動化測試框架，因此本文件的案例應搭配頁面結果、可展開乘區明細與 Formula Debug 一起人工核對。

## 2. 測試前提

- V1 是純靜態網站，應能直接由 GitHub Pages 從 repository root 部署。
- V1 先支援大黑塔單次傷害計算，不包含完整回合軸、行動序或能量循環。
- 技能倍率以目前 `data/characters/the-herta.json` 為準。
- 光錐效果以目前 `data/light-cones.json` 為準。
- 若遊戲資料仍不確定，請標記為 TODO，不要臆測套用。
- 目前測試重點是 normal、crit、expected、per-hit breakdown、乘區摘要與 Formula Debug 是否一致。

## 3. 基礎 Smoke Test

手動檢查項目：

- 網站可以正常開啟。
- Browser Console 沒有錯誤。
- 切換技能時，傷害結果會更新。
- 切換光錐時，攻擊力、暴率、暴傷或增傷會依目前支援的效果更新。
- 結果區、乘區明細與 Formula Debug 不應出現 `NaN`、`Infinity` 或空白。
- 乘區係數每個項目都應可展開，並能看到公式與來源明細。
- 乘區係數應顯示 `傷害乘區總值` 與 `完整期望倍率`。

## 4. 大黑塔技能倍率測試

目前 V1 應檢查下列技能倍率：

- 普攻：指定單體 `100%`。
- 戰技：指定單體 `70%`；命中目標及相鄰目標重複 `2` 次，每段 `70%`。
- 強化戰技：指定單體 `80%`；命中目標及相鄰目標重複 `2` 次，每段 `80%`；敵方全體收尾 `40%`。
- 終結技：敵方全體 `200%`。
- V1 不應預設套用 6 魂終結技額外倍率 `140%` / `250%` / `400%`。

建議核對方式：

- 切換每個技能後，確認 Formula Debug 或 per-hit breakdown 顯示的 hit 倍率、目標數與重複次數符合上述資料。
- 終結技測試時，確認 Debug 有標示 V1 未套用 6 魂額外倍率。

## 5. 解讀層數測試

### 強化戰技，解讀層數 0

設定：

- 技能：強化戰技。
- 解讀層數：`0`。

預期：

- 不應出現額外解讀倍率。
- 主目標、其他目標與收尾 hit 只應反映技能基礎倍率與其他已啟用乘區。

### 強化戰技，解讀層數 42，未啟用 2 智識

設定：

- 技能：強化戰技。
- 解讀層數：`42`。
- 2 智識條件：未啟用。

預期：

- 主目標每層 `+8%`。
- 其他目標每層 `+4%`。
- Formula Debug 或 per-hit breakdown 應能看出解讀層數倍率來源。

### 強化戰技，解讀層數 42，啟用 2 智識

設定：

- 技能：強化戰技。
- 解讀層數：`42`。
- 2 智識條件：啟用。

預期：

- 主目標每層 `+16%`。
- 其他目標每層 `+8%`。
- Formula Debug 或 per-hit breakdown 應能看出 2 智識條件造成解讀倍率提高。

### 舊倍率回歸檢查

確認目前不再出現舊倍率：

- 不應出現主目標 `20%` / 其他目標 `10%`。
- 不應出現主目標 `10%` / 其他目標 `5%`。

## 6. 光錐效果測試

### A. none

預期：

- `baseAtk` 為 `0`。
- 不提供暴率。
- 不提供暴傷。
- 不提供增傷。

### B. signature：向著不可追問處

預期：

- `baseAtk` 為 `635`。
- 暴率 `+12%`。
- `afterUltimate` 未啟用時，不給戰技 / 終結技 `+60%` 增傷。
- `afterUltimate` 啟用時，戰技與終結技獲得 `+60%` 增傷。
- 普攻不吃 `afterUltimate` 的 `+60%` 增傷。

建議核對方式：

- 分別測試普攻、戰技、強化戰技與終結技。
- 分別切換 `afterUltimate` 關閉與啟用。
- 檢查乘區摘要或 Formula Debug 中的光錐暴率與光錐增傷來源。

### C. genius：拂曉之前

預期：

- `baseAtk` 為 `582`。
- 暴傷 `+36%`。
- 戰技與終結技 `+18%` 增傷。
- 追擊增傷資料可以存在，但 V1 沒有追擊技能時不應套用。

建議核對方式：

- 測試普攻時，不應吃戰技 / 終結技增傷。
- 測試戰技、強化戰技與終結技時，應吃對應增傷。
- 檢查 Formula Debug 中的光錐暴傷與光錐增傷來源。

### D. cosmos：銀河鐵道之夜

預期：

- `baseAtk` 為 `582`。
- 條件效果目前 TODO。
- 不應臆測套用尚未確認的條件效果。

## 7. 敵人乘區測試

手動檢查項目：

- 敵人等級改變時，防禦區會變動。
- 防禦降低增加時，防禦乘區應提高。
- 冰抗增加時，抗性乘區應降低。
- 抗性穿透增加時，抗性乘區應提高。
- 易傷增加時，易傷區應提高。
- 減傷增加時，最終傷害應降低。
- 弱點相關設定變更時，弱點區應反映在乘區摘要或 Debug。

## 8. Debug / 乘區來源測試

目前應該能在可展開乘區明細或 Formula Debug 中看到下列來源：

- 光錐暴率。
- 光錐暴傷。
- 光錐增傷。
- 大黑塔終結技後攻擊 Buff。
- 解讀層數倍率。
- 謎底層數倍率。
- V1 未套用 6 魂終結技額外倍率。

檢查時應確認來源文字與實際套用效果一致；若只顯示結果但看不出來源，應記錄為後續 Debug 改善項目。

### 可展開乘區明細

手動檢查項目：

- 攻擊區展開後應顯示基礎攻擊、攻擊%、固定攻擊與大黑塔終結技後攻擊 Buff。
- 增傷區展開後應顯示主詞條、行跡、光錐、套裝與 42 層解讀行跡增傷等目前生效來源。
- 暴擊期望展開後應顯示計算用暴率、總暴傷與暴率/暴傷來源。
- 防禦區、抗性區、易傷區、我方減傷區、敵方減傷區與弱點區展開後應顯示目前敵人或狀態輸入。
- 切換光錐後，展開明細應反映目前光錐來源。
- `afterUltimate` 開關應影響向著不可追問處在戰技 / 終結技的增傷明細。
- `傷害乘區總值` 應不含攻擊與技能倍率。
- `完整期望倍率` 應不含攻擊力，並包含技能倍率區。

## 9. Regression Checklist

每次修改公式、資料或相關 UI 後，至少檢查：

- 網站能正常載入。
- JSON 能正常載入。
- 沒有 `NaN` / `Infinity`。
- 技能切換正常。
- 光錐切換正常。
- `afterUltimate` 切換正常。
- `twoErudition` 切換正常。
- 公式 Debug 或乘區明細能看出主要來源。
- 不應意外套用未支援的星魂、光錐條件或角色機制。
