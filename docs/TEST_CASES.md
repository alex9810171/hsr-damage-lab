# V1 Manual Test Cases

本文記錄 HSR Damage Lab V1 單次傷害計算的手動測試案例，用來確保未來修改公式、光錐、隊伍 Buff、敵人乘區或 UI 時，可以檢查輸出是否仍合理。

## 1. 文件目的

這份文件用來保存 V1 的人工回歸測試清單。每次修改傷害公式、資料來源、光錐效果、隊伍 Buff、敵人乘區、Formula Debug 或 UI 顯示時，應依照本文件檢查主要輸出是否仍符合目前設計。

目前尚未接入自動化測試框架，因此本文件的案例應搭配頁面結果與可展開乘區明細人工核對。Formula Debug 保留在程式中作為開發工具，但不在主要 UI 顯示。

## 2. 測試前提

- V1 是純靜態網站，應能直接由 GitHub Pages 從 repository root 部署。
- V1 先支援大黑塔單次傷害計算，不包含完整回合軸、行動序或能量循環。
- 技能倍率以目前 `data/characters/the-herta.json` 為準。
- 光錐效果以目前 `data/light-cones.json` 為準。
- 若遊戲資料仍不確定，請標記為 TODO，不要臆測套用。
- 目前測試重點是 normal、crit、expected、技能目標總覽與乘區摘要是否一致。

## 3. 基礎 Smoke Test

手動檢查項目：

- 網站可以正常開啟。
- Browser Console 沒有錯誤。
- 切換技能時，傷害結果會更新。
- 切換光錐時，攻擊力、暴率、暴傷或增傷會依目前支援的效果更新。
- 結果區與乘區明細不應出現 `NaN`、`Infinity` 或空白。
- 乘區係數每個項目都應可展開，並能看到公式與來源明細。
- 乘區係數應顯示 `攻擊區`、`技能總倍率`、`通用乘區倍率` 與 `不含攻擊力期望倍率`。
- 乘區係數應顯示 `相對基準總倍率`，並可展開查看各區相對倍率構成。
- 主結果卡片應顯示期望傷害、暴擊傷害與非暴擊傷害；技能總倍率與不含攻擊力期望倍率改放在技能拆解 / 乘區係數中。
- 攻擊區展開後應顯示攻擊力倍率，且 `攻擊力倍率 = 最終攻擊力 / 基礎攻擊力`。最終攻擊力本身不應以 `x` 倍顯示。
- 技能拆解可以展開 / 收合，目標總覽預設展開，詳細 hit 計算預設收合。
- 技能拆解的目標總覽應能展開每個目標，查看該目標倍率由哪些命中列構成。
- 技能總倍率摘要不應預設顯示超長 hit 公式。
- 邊際提升參考應顯示為單步配置比較，不暗示最佳化器。
- 邊際提升參考應說明它以目前完整配置作為 `1.0000x` 基準；防禦降低 +10% 這類項目應顯示相對提升，不是直接把防禦區加 10%。
- 邊際提升參考不應出現弱點區候選項；弱點區仍應保留於實際公式乘區與相對基準總倍率。
- 手動修改隊友欄位後，隊伍摘要應依實際欄位更新；隊伍 preset 不再作為主要 UI 控制。

## 4. 大黑塔技能倍率測試

目前 V1 應檢查下列技能倍率：

- 普攻：指定單體 `100%`。
- 戰技 5 敵總分布：左二 `70%`、左一 `140%`、主目標 `210%`、右一 `140%`、右二 `70%`。
- 強化戰技 5 敵基礎列：主目標 `80%`；主目標及相鄰 `80%`；敵方全體 `80%`；敵方全體收尾 `40%`。
- 終結技：敵方全體 `200%`。
- V1 不應預設套用 6 魂終結技額外倍率 `140%` / `250%` / `400%`。

建議核對方式：

- 切換每個技能後，確認 Formula Debug 或 per-hit breakdown 顯示的 hit 倍率、目標數與重複次數符合上述資料。
- 終結技測試時，確認 Debug 有標示 V1 未套用 6 魂額外倍率。

## 5. 解讀層數測試

### 強化戰技，解讀層數 0

設定：

- 技能：強化戰技。
- 解讀層數：`0`。

預期：

- 不應出現額外解讀倍率。
- 主目標、其他目標與收尾 hit 只應反映技能基礎倍率與其他已啟用乘區。

### 強化戰技，解讀層數 42，未啟用 2 智識

設定：

- 技能：強化戰技。
- 解讀層數：`42`。
- 2 智識條件：未啟用。

預期：

- 主目標每層 `+8%`。
- 其他目標每層 `+4%`。
- Formula Debug 或 per-hit breakdown 應能看出解讀層數倍率來源。

### 強化戰技，解讀層數 42，啟用 2 智識

設定：

- 技能：強化戰技。
- 解讀層數：`42`。
- 2 智識條件：啟用。

預期：

- 主目標每層 `+16%`。
- 其他目標每層 `+8%`。
- Formula Debug 或 per-hit breakdown 應能看出 2 智識條件造成解讀倍率提高。
- 5 敵時解讀列應為 `336 / 336 / 672 / 336 / 336`。

### 強化戰技，解讀層數 42，啟用 2 智識，啟用一魂

設定：

- 技能：強化戰技。
- 解讀層數：`42`。
- 2 智識條件：啟用。
- 大黑塔一魂解讀倍率：啟用。

預期：

- 一魂只影響強化戰技解讀列，不啟用其他星魂效果。
- 5 敵時解讀列應為 `504 / 504 / 1008 / 504 / 504`。
- 一魂預設應為關閉；只有手動勾選後才套用。

### 舊倍率回歸檢查

確認目前不再出現舊倍率：

- 不應出現主目標 `20%` / 其他目標 `10%`。
- 不應出現主目標 `10%` / 其他目標 `5%`。

## 6. 光錐效果測試

### A. none

預期：

- `baseAtk` 為 `0`。
- 不提供暴率。
- 不提供暴傷。
- 不提供增傷。

### B. signature：向著不可追問處

預期：

- `baseAtk` 為 `635`。
- 暴率 `+12%`。
- `afterUltimate` 未啟用時，不給戰技 / 終結技 `+60%` 增傷。
- `afterUltimate` 啟用時，戰技與終結技獲得 `+60%` 增傷。
- 普攻不吃 `afterUltimate` 的 `+60%` 增傷。

建議核對方式：

- 分別測試普攻、戰技、強化戰技與終結技。
- 分別切換 `afterUltimate` 關閉與啟用。
- 檢查乘區摘要或 Formula Debug 中的光錐暴率與光錐增傷來源。

### C. genius：拂曉之前

預期：

- `baseAtk` 為 `582`。
- 暴傷 `+36%`。
- 戰技與終結技 `+18%` 增傷。
- 追擊增傷資料可以存在，但 V1 沒有追擊技能時不應套用。

建議核對方式：

- 測試普攻時，不應吃戰技 / 終結技增傷。
- 測試戰技、強化戰技與終結技時，應吃對應增傷。
- 檢查 Formula Debug 中的光錐暴傷與光錐增傷來源。

### D. cosmos：銀河鐵道之夜

預期：

- `baseAtk` 為 `582`。
- 條件效果目前 TODO。
- 不應臆測套用尚未確認的條件效果。

## 7. 敵人乘區測試

手動檢查項目：

- 敵人等級改變時，防禦區會變動。
- 防禦降低增加時，防禦乘區應提高。
- 冰抗增加時，抗性乘區應降低。
- 抗性穿透增加時，抗性乘區應提高。
- 易傷增加時，易傷區應提高。
- 減傷增加時，最終傷害應降低。
- 弱點相關設定變更時，弱點區應反映在乘區摘要或 Debug。

## 8. Debug / 乘區來源測試

目前應該能在可展開乘區明細或 Formula Debug 中看到下列來源：

- 光錐暴率。
- 光錐暴傷。
- 光錐增傷。
- 大黑塔終結技後攻擊 Buff。
- 解讀層數倍率。
- 謎底層數倍率。
- V1 未套用 6 魂終結技額外倍率。

檢查時應確認來源文字與實際套用效果一致；若只顯示結果但看不出來源，應記錄為後續 Debug 改善項目。

### 可展開乘區明細

手動檢查項目：

- 攻擊區展開後應顯示基礎攻擊、攻擊%、固定攻擊與大黑塔終結技後攻擊 Buff。
- 增傷區展開後應顯示主詞條、行跡、光錐、套裝與 42 層解讀行跡增傷等目前生效來源。
- 暴擊期望展開後應顯示計算用暴率、總暴傷與暴率/暴傷來源。
- 防禦區、抗性區、易傷區、我方減傷區、敵方減傷區與弱點區展開後應顯示目前敵人或狀態輸入。
- 防禦區展開後應標示實際防禦乘區會直接進入傷害公式，並顯示防禦相對倍率 `目前防禦區 / 基準防禦區`。
- 弱點區展開後應標示它屬於戰鬥狀態乘區，可納入相對基準總倍率，但不列入邊際提升參考。
- 切換光錐後，展開明細應反映目前光錐來源。
- `afterUltimate` 開關應影響向著不可追問處在戰技 / 終結技的增傷明細。
- `通用乘區倍率` 應不含攻擊力與技能倍率。
- `不含攻擊力期望倍率` 應等於技能總倍率 x 通用乘區倍率。
- `期望傷害` 應等於最終攻擊力 x 不含攻擊力期望倍率。
- `相對基準總倍率` 不應改變 `期望傷害`、`暴擊傷害` 或 `非暴擊傷害`。
- 邊際提升參考不是 optimizer，不應使用暗示已完成配裝搜尋或自動配裝建議的文案。

### 技能倍率顯示

手動檢查項目：

- 乘區係數中的 `技能總倍率` 展開後應顯示各目標倍率總覽與加總公式。
- 技能總倍率應等於各目標倍率總和。
- 普通戰技 5 敵總覽仍是 `70 / 140 / 210 / 140 / 70`。
- 強化戰技 5 敵、42 解讀、2 智識、無一魂總覽仍是 `456 / 536 / 952 / 536 / 456`。
- 強化戰技 5 敵、42 解讀、2 智識、有一魂總覽仍是 `624 / 704 / 1288 / 704 / 624`。
- 目標總覽展開後應看到基礎命中列、解讀層數倍率與合計，詳細 hit 計算仍預設收合。

## 9. Regression Checklist

每次修改公式、資料或相關 UI 後，至少檢查：

- 網站能正常載入。
- JSON 能正常載入。
- 沒有 `NaN` / `Infinity`。
- 技能切換正常。
- 光錐切換正常。
- `afterUltimate` 切換正常。
- `twoErudition` 切換正常。
- `hertaE1` 預設關閉，切換後只影響強化戰技解讀列。
- 公式 Debug 或乘區明細能看出主要來源。
- 不應意外套用未支援的星魂、光錐條件或角色機制。

- 主要畫面沒有造成誤解的隊伍預設下拉、終結技後攻擊 +80% 手動開關或 2 智識手動開關。
- 預設隊伍為大黑塔 / 那刻夏 / 緹寶 / 風堇；隊伍摘要會依實際隊友欄位更新。

- 大黑塔終結技後攻擊 +80% 預設常駐，攻擊區顯示來源。
