# Decisions

本文記錄目前已確定的架構決策。若後續決策改變，請新增日期與原因，不要直接刪除舊脈絡。

## 2026-06-07

### V1 維持純靜態網站

目前專案維持 `index.html`、`assets/css/styles.css`、`assets/js/*.js`
與 `data/*.json` 的純靜態結構。網站應可直接由 GitHub Pages 從
repository root 部署。

### 暫時不改 React

V1 不引入 React、Vite、TypeScript、npm build step 或其他前端框架。這能降低部署與維護成本，也符合目前 GitHub Pages 靜態部署方式。

### 不即時爬取 Nanoka 資料

角色、光錐、遺器與隊伍資料先以本地 JSON 維護。未來若要參考外部資料來源，也應先轉成可審查的本地資料，不在前端執行時即時爬取。

### 先做單次傷害，不做回合軸

第一版聚焦大黑塔單次傷害計算，包含技能 hit、暴擊/非暴擊/期望傷害、敵人乘區與必要角色特殊機制。暫時不做行動序、Buff 持續時間、能量循環或完整戰鬥模擬。

### 先補文件，再逐步拆分 calculator.js

目前 `assets/js/calculator.js` 同時包含狀態彙整、通用傷害公式、敵人乘區、部分隊伍 Buff 與大黑塔特殊機制。短期先補齊文件與資料模型方向，後續再小步拆分。

### calculator.js 未來作為協調者

長期方向是讓 `calculator.js` 負責協調流程，而不是塞滿所有公式。建議逐步拆出：

- `damage-engine.js`：通用傷害公式與乘區。
- `skill-engine.js`：技能與 hit 解析。
- `buff-engine.js`：Buff 彙整。
- `character-resolvers/*.js`：角色特殊機制。

### 允許角色特殊機制，但必須集中管理

大黑塔的解讀層數、謎底層數、終結技攻擊力提高、智識隊友條件與依敵人數
改變倍率等機制可以存在，但應集中管理。不要讓同一角色規則散落在
`ui.js`、`calculator.js`、`formula-debug.js` 與未來的 engine 檔案中。

### UI 不應硬寫角色資料或公式

UI 可以負責呈現欄位、讀取輸入與渲染結果，但不應直接硬寫角色技能倍率、角色特殊公式、Buff 規則或敵人乘區。若目前 UI 中存在 preset 或呈現文字，後續應逐步改由資料或計算結果驅動。

## 目前觀察到的落差

- `calculator.js` 目前仍包含多種職責，和未來「協調者」目標不同。
- 大黑塔部分特殊機制目前仍在 `calculator.js` 內處理，未集中到 `skill-engine.js` 或角色 resolver。
- `formula-debug.js` 目前輸出公式細節，未來拆分公式時需要同步保持 debug 文字與計算來源一致。
- `ui.js` 目前包含 preset 套用與隊伍欄位同步，後續若支援多角色，部分邏輯可能需要資料化或移到資料解析層。
- 現有 `README.md` 與既有 docs 在目前終端環境顯示為亂碼；本次不修改既有文件內容，以避免破壞原始編碼。

## 2026-06-08 V1 隊伍與星魂整理

### 隊伍 UI 改由實際欄位驅動

「隊伍預設」不再作為主要 UI 控制，只保留為初始化資料。畫面顯示目前隊伍摘要：
大黑塔 / 那刻夏 / 緹寶 / 風堇，並由實際隊友欄位即時生成。計算端以隊友欄位的角色、
星魂、光錐、Cavern 與 Planar 為準。

### 預設副詞條

V1 預設副詞條改為暴擊率 10、暴擊傷害 17、攻擊% 3、固定攻擊 0，總詞條數 30。
每條副詞條數值不變。

### 大黑塔終結技攻擊 Buff 常駐

大黑塔終結技後攻擊力 +80% 在 V1 單次傷害中視為常駐，不再提供手動開關。攻擊區需顯示
「大黑塔終結技後攻擊力 +80%（V1 預設常駐）」；V1 不處理 3 回合持續時間與回合軸。

### 2 智識自動判斷

2 智識條件由隊伍命途自動判斷。大黑塔與那刻夏都是智識，因此預設隊伍智識數量為 2，
會自動啟用大黑塔 2 智識解讀倍率與那刻夏「必要的留白」全隊造成傷害 +50%。

### 大黑塔星魂

大黑塔星魂改由 0～6 魂按鈕控制，預設 0 魂。

- 1 魂：V1 以有效解讀層數 = 實際解讀層數 x 1.5 處理，會影響 42 層行跡增傷判斷。
- 2 魂：靈感 / 行動提前屬於行動機制，V1 只展示，不直接加入單次傷害乘區。
- 3 魂：以資料核對結果記錄為技能等級 +2、天賦等級 +2；因缺少可靠 Lv12/Lv15 倍率表，V1 只展示 TODO，不改倍率。
- 4 魂：智識角色速度 +12%，屬於速度 / 回合軸效果，V1 只展示。
- 5 魂：以資料核對結果記錄為終結技等級 +2、普攻等級 +1；因缺少可靠 Lv12/Lv15 倍率表，V1 只展示 TODO，不改倍率。
- 6 魂：已支援冰屬性抗性穿透 +20%；終結技在敵人數 >=3 / 2 / 1 時分別追加倍率 +140% / +250% / +400%。

### 技能拆解與 Debug

主結果卡片只保留期望、暴擊、非暴擊傷害。技能總倍率主要放在技能拆解；乘區係數中的技能總倍率
只保留各目標摘要。Formula Debug 不在主要 UI 顯示，檔案保留作為開發工具。

### 指定隊伍效果

那刻夏「必要的留白」在 2 智識以上時提供全隊造成傷害 +50%，套用到增傷區；那刻夏 0 魂不套用
E1 降防，多弱點狀態先標 TODO。緹寶 +24% 全屬性抗性穿透與 +30% 受傷提高已套用；緹寶 1 魂
真實傷害以大黑塔本體總傷害 x 24% 獨立加入結果，不再次吃大黑塔乘區。舞！舞！舞！只展示行動提前。
風堇光錐愿虹光永駐天空 +18% 受傷提高已套用；風堇本體其他效果暫不套用。

## 2026-06-07 乘區明細顯示

### factorSummary 作為乘區明細資料來源

`factorSummary` 現在支援 `label`、`value`、`formula`、`detail` 與 `parts`。
乘區來源明細由 `calculator.js` 產生，`ui.js` 只負責以 `details` / `summary`
渲染，不在 UI 重新計算公式。

### 新增乘區總值

乘區係數區塊使用三個明確名稱：

- 技能總倍率：目前技能在目前敵人數與命中分布下的 `Σ(技能倍率 x 目標數 x 次數)`。
- 通用乘區倍率：增傷區、暴擊期望、防禦區、抗性區、易傷區、我方減傷區、敵方減傷區與弱點區相乘，不含攻擊力與技能倍率。
- 不含攻擊力期望倍率：技能總倍率乘上通用乘區倍率。

避免再使用容易誤導的「傷害乘區總值」或「完整期望倍率」作為 UI 名稱。期望傷害才是最終攻擊力乘上不含攻擊力期望倍率。

### Formula Debug 暫時保留

Formula Debug 仍保留作為較完整的文字追蹤。主要閱讀入口改為可展開的乘區明細；
未來可逐步把重複內容收斂到同一份結構化來源。

## 2026-06-08 V1 UI 與技能分布修正

### 隊伍預設必須同步實際隊友欄位

隊伍預設現在只代表目前套用到隊友欄位的配置。使用者選擇 preset 時會同步隊友欄位；
手動修改任一隊友、星魂或裝備效果欄位時，隊伍預設會改為「自訂隊伍」。V1 自動隊伍
Buff 以目前隊友欄位為準，避免 UI 顯示某個 preset 但計算套用其他隊伍。

### V1 不做最佳化器

V1 只提供目前配置下的單步配置比較與乘區參考，不執行配裝搜尋，也不宣稱能產生自動化配裝建議。
完整最佳化器屬於後續版本，需另行設計資料驗證、限制條件與比較目標。

### 主結果卡片以傷害與乘區為主

主結果卡片改顯示期望傷害、暴擊傷害、技能總倍率與不含攻擊力期望倍率。最終攻擊力仍保留
在輔助資訊卡與攻擊區展開明細中，但不再作為主結果。

### 技能拆解改為折疊式閱讀

技能拆解使用 `details` / `summary` 分組。目標總覽預設展開，作為主要閱讀入口；基礎命中列、解讀層數倍率與詳細 hit 計算預設收合，避免結果頁一開始被所有 hit row 撐長。

### 邊際提升參考不是 optimizer

原本的配置比較參考改名為「邊際提升參考」，放在非暴擊傷害與最終攻擊力之後，作為附屬提示。文案維持「這只是以當前面板做單步配置比較，不代表已執行最佳化器」，並標示僅比較單一變數變動。

### 大黑塔技能改用命中分布理解

大黑塔戰技與強化戰技改用命中列分布表示。5 敵普通戰技總分布為
`70 / 140 / 210 / 140 / 70`。強化戰技在 5 敵時保留基礎列與解讀列拆解，
42 解讀、2 智識時無一魂解讀列為 `336 / 336 / 672 / 336 / 336`，一魂開啟後為
`504 / 504 / 1008 / 504 / 504`。

### 一魂開關範圍

V1 新增「大黑塔一魂解讀倍率」開關，預設關閉。此開關只影響強化戰技解讀列的等效
`50%` 提升，不加入其他星魂效果，也不重新加入 6 魂終結技額外倍率。

### 遺器與位面飾品支援狀態

遺器與位面飾品選項需標示「已支援」「部分支援」「待核對」。未確認或未完整支援的效果
不自動套用；資料應以 `supportStatus`、`notes` 或 TODO 標記，逐步核對後再接入。

### 實際公式乘區與相對基準總倍率分流

V1 新增 `相對基準總倍率` 作為輔助理解值，但不取代實際公式乘區。實際防禦區仍可能顯示像 `0.4651x` 這種直接進入傷害公式的乘區；防禦相對倍率則是 `目前防禦區 / 基準防禦區`。

相對基準總倍率由攻擊力倍率、增傷相對倍率、暴擊期望倍率、防禦相對倍率、抗性相對倍率、易傷相對倍率、我方減傷相對倍率、敵方減傷相對倍率與弱點相對倍率相乘。它只用於理解目前配置相對基準狀態的倍率，不應改變期望傷害、暴擊傷害或非暴擊傷害。

攻擊力倍率定義為 `最終攻擊力 / 基礎攻擊力`。最終攻擊力仍是傷害公式使用的基礎數值，不用 `x` 倍顯示。

弱點區保留在實際公式與相對基準總倍率中，因為它是完整戰鬥狀態的一部分；但不列入邊際提升參考，避免讓使用者誤以為它是單純配裝或 Buff 可控項。

邊際提升參考繼續以目前完整配置作為 `1.0000x`，用 `新期望傷害 / 原期望傷害 - 1` 表示相對提升。例如防禦降低 +10% 若讓防禦區由 `0.4651x` 變成 `0.4914x`，應解讀為 `0.4914 / 0.4651 - 1 = +5.65%`。

### 技能倍率閱讀入口

技能總倍率明細以各目標倍率總覽與加總公式為主，避免在 summary 直接塞入冗長 hit 公式。技能拆解的目標總覽可展開單一目標，查看該目標由哪些基礎命中列與解讀層數倍率構成；技能總倍率仍等於各目標倍率總和。
