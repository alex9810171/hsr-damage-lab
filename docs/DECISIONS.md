# Decisions

本文記錄目前已確定的架構決策。若後續決策改變，請新增日期與原因，不要直接刪除舊脈絡。

## 2026-06-07

### V1 維持純靜態網站

目前專案維持 `index.html`、`assets/css/styles.css`、`assets/js/*.js`
與 `data/*.json` 的純靜態結構。網站應可直接由 GitHub Pages 從
repository root 部署。

### 暫時不改 React

V1 不引入 React、Vite、TypeScript、npm build step 或其他前端框架。這能降低部署與維護成本，也符合目前 GitHub Pages 靜態部署方式。

### 不即時爬取 Nanoka 資料

角色、光錐、遺器與隊伍資料先以本地 JSON 維護。未來若要參考外部資料來源，也應先轉成可審查的本地資料，不在前端執行時即時爬取。

### 先做單次傷害，不做回合軸

第一版聚焦大黑塔單次傷害計算，包含技能 hit、暴擊/非暴擊/期望傷害、敵人乘區與必要角色特殊機制。暫時不做行動序、Buff 持續時間、能量循環或完整戰鬥模擬。

### 先補文件，再逐步拆分 calculator.js

目前 `assets/js/calculator.js` 同時包含狀態彙整、通用傷害公式、敵人乘區、部分隊伍 Buff 與大黑塔特殊機制。短期先補齊文件與資料模型方向，後續再小步拆分。

### calculator.js 未來作為協調者

長期方向是讓 `calculator.js` 負責協調流程，而不是塞滿所有公式。建議逐步拆出：

- `damage-engine.js`：通用傷害公式與乘區。
- `skill-engine.js`：技能與 hit 解析。
- `buff-engine.js`：Buff 彙整。
- `character-resolvers/*.js`：角色特殊機制。

### 允許角色特殊機制，但必須集中管理

大黑塔的解讀層數、謎底層數、終結技攻擊力提高、智識隊友條件與依敵人數
改變倍率等機制可以存在，但應集中管理。不要讓同一角色規則散落在
`ui.js`、`calculator.js`、`formula-debug.js` 與未來的 engine 檔案中。

### UI 不應硬寫角色資料或公式

UI 可以負責呈現欄位、讀取輸入與渲染結果，但不應直接硬寫角色技能倍率、角色特殊公式、Buff 規則或敵人乘區。若目前 UI 中存在 preset 或呈現文字，後續應逐步改由資料或計算結果驅動。

## 目前觀察到的落差

- `calculator.js` 目前仍包含多種職責，和未來「協調者」目標不同。
- 大黑塔部分特殊機制目前仍在 `calculator.js` 內處理，未集中到 `skill-engine.js` 或角色 resolver。
- `formula-debug.js` 目前輸出公式細節，未來拆分公式時需要同步保持 debug 文字與計算來源一致。
- `ui.js` 目前包含 preset 套用與隊伍欄位同步，後續若支援多角色，部分邏輯可能需要資料化或移到資料解析層。
- 現有 `README.md` 與既有 docs 在目前終端環境顯示為亂碼；本次不修改既有文件內容，以避免破壞原始編碼。

## 2026-06-07 V1 傷害修正

### 大黑塔技能倍率校正

V1 暫以 Lv10 技能 / 普攻滿級作為預設，已校正大黑塔普攻、戰技、強化戰技與
終結技倍率。強化戰技解讀層數改為沒有 2 智識時主目標 `+8%`、其他目標
`+4%`；有 2 智識時主目標 `+16%`、其他目標 `+8%`。

### 不預設套用 6 魂終結技額外倍率

V1 移除大黑塔終結技依敵人數量預設追加 `140%` / `250%` / `400%` 的行為。
若未來支援 6 魂，應由明確的 eidolon 狀態或資料欄位啟用，不作為 V1 預設。

### 光錐效果先納入 V1 必要項目

`data/light-cones.json` 已加入向著不可追問處與拂曉之前的 V1 可用效果：
暴擊率、暴擊傷害、戰技增傷、終結技增傷，以及向著不可追問處的終結技後增傷。
銀河鐵道之夜目前只使用基礎攻擊力，條件效果保留 TODO。

### Debug 展開留待後續

本次只小幅補強 formula debug，使其能看出光錐來源、大黑塔終結技後攻擊 Buff、
解讀層數、謎底層數，以及 V1 未套用 6 魂終結技額外倍率。
完整的乘區展開明細與公式 Debug 合併留到後續任務。

## 2026-06-07 乘區明細顯示

### factorSummary 作為乘區明細資料來源

`factorSummary` 現在支援 `label`、`value`、`formula`、`detail` 與 `parts`。
乘區來源明細由 `calculator.js` 產生，`ui.js` 只負責以 `details` / `summary`
渲染，不在 UI 重新計算公式。

### 新增乘區總值

乘區係數區塊新增兩個總值：

- 傷害乘區總值：增傷區、暴擊期望、防禦區、抗性區、易傷區、我方減傷區、敵方減傷區與弱點區相乘，不含攻擊與技能倍率。
- 完整期望倍率：倍率區乘上傷害乘區總值，不含攻擊力。

多段技能的倍率區先使用目前結果中的 `Σ(技能倍率 x 目標數 x 次數)`。

### Formula Debug 暫時保留

Formula Debug 仍保留作為較完整的文字追蹤。主要閱讀入口改為可展開的乘區明細；
未來可逐步把重複內容收斂到同一份結構化來源。

## 2026-06-08 V1 UI 與技能分布修正

### 隊伍預設必須同步實際隊友欄位

隊伍預設現在只代表目前套用到隊友欄位的配置。使用者選擇 preset 時會同步隊友欄位；
手動修改任一隊友、星魂或裝備效果欄位時，隊伍預設會改為「自訂隊伍」。V1 自動隊伍
Buff 以目前隊友欄位為準，避免 UI 顯示某個 preset 但計算套用其他隊伍。

### V1 不做最佳化器

V1 只提供目前配置下的單步配置比較與乘區參考，不執行配裝搜尋，也不宣稱能產生自動化配裝建議。
完整最佳化器屬於後續版本，需另行設計資料驗證、限制條件與比較目標。

### 主結果卡片以傷害與乘區為主

主結果卡片改顯示期望傷害、暴擊傷害、傷害乘區總值與完整期望倍率。最終攻擊力仍保留
在輔助資訊卡與攻擊區展開明細中，但不再作為主結果。

### 大黑塔技能改用命中分布理解

大黑塔戰技與強化戰技改用命中列分布表示。5 敵普通戰技總分布為
`70 / 140 / 210 / 140 / 70`。強化戰技在 5 敵時保留基礎列與解讀列拆解，
42 解讀、2 智識時無一魂解讀列為 `336 / 336 / 672 / 336 / 336`，一魂開啟後為
`504 / 504 / 1008 / 504 / 504`。

### 一魂開關範圍

V1 新增「大黑塔一魂解讀倍率」開關，預設關閉。此開關只影響強化戰技解讀列的等效
`50%` 提升，不加入其他星魂效果，也不重新加入 6 魂終結技額外倍率。

### 遺器與位面飾品支援狀態

遺器與位面飾品選項需標示「已支援」「部分支援」「待核對」。未確認或未完整支援的效果
不自動套用；資料應以 `supportStatus`、`notes` 或 TODO 標記，逐步核對後再接入。
